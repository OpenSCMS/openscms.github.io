Frequently Asked Questions

Q. What is OpenSCMS?

OpenSCMS is an open-source, server-side implementation of the core components defined by the IEEE 1609.2.1 Secure Credential Management System (SCMS). It provides certificate lifecycle management, cryptographic services, and protocol handling for V2X ecosystems.

Q. Is OpenSCMS compliant with IEEE 1609.2.1?

Yes. OpenSCMS implements the core server-side flows and data structures defined by IEEE 1609.2.1, including enrollment, authorization, certificate distribution, and trust artifact handling. Validation is performed using unit tests and end-to-end testing with a certified V2X client.

Q. Does OpenSCMS include a V2X client SDK?

No. OpenSCMS is a server-side implementation only. It is designed to interoperate with external V2X clients that implement IEEE 1609.2.1. Client SDKs are outside the scope of the project.

Q. Which SCMS components are implemented?

OpenSCMS v1.0.0 includes the Registration Authority (RA), Enrollment Certificate Authority (ECA), Authorization Certificate Authority (ACA), Linkage Authority (LA), and certificate and trust distribution services such as CTL, CCF, CRL, and CMIS.

Q. What types of certificates are supported?

OpenSCMS supports both explicit and implicit certificates as defined by IEEE 1609.2.1. Implicit certificates reduce payload size and are supported across enrollment and authorization flows.

Q. Which butterfly key mechanisms are supported?

The supported butterfly key mechanisms are Original Butterfly Keys (OBK), Unified Butterfly Keys (UBK), and Compact Unified Butterfly Keys (CUBK). Both encrypted and non-encrypted flows are supported.

Q. How are cryptographic operations implemented?

All cryptographic operations and ASN.1 encoding/decoding are handled by the standalone C library oscms-codecs-bridge, which uses OpenSSL. The library is integrated into the Rust backend using automatically generated bindings.

Q. Why does OpenSCMS use both Rust and C?

Rust is used for service orchestration, concurrency, and API handling, while C is used for the cryptographic and ASN.1 codec core to ensure performance and precise control. The integration between the two is transparent for service developers.

Q. How can I deploy OpenSCMS?

OpenSCMS can be deployed using a unified Docker + Kubernetes + Skaffold setup (recommended), or manually by running individual components with externally managed dependencies such as MySQL and RabbitMQ.

Q. Is OpenSCMS production-ready?

Version 1.0.0 is stable and suitable for development, testing, and evaluation. Production deployments require environment-specific hardening and operational tooling that are outside the scope of the core project.
