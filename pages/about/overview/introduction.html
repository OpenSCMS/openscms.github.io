Background

Introduction

Autonomous and connected vehicles depend on secure and reliable V2X (V2V/V2I) communication to enable cooperative awareness, coordinated maneuvers, traffic optimization, and improved road safety. In this context, security and privacy are foundational.

The IEEE 1609.2 and IEEE 1609.2.1 standards define the security architecture for V2X. At its core is the Security Credential Management System (SCMS), a PKI responsible for issuing and managing digital certificates for V2X end entities (e.g., OBUs and RSUs). These certificates encode application permissions via PSIDs and SSPs (IEEE 1609.12).

A key SCMS goal is privacy: vehicles must be able to authenticate messages without being trackable over time. This is typically achieved with pseudonym certificates that are rotated regularly.

Despite its importance, SCMS is difficult to implement: it is distributed, performance-sensitive, and operationally complex. Most existing implementations are proprietary, hard to audit, and expensive to extend—limiting innovation and research.

OpenSCMS addresses this gap with an open-source implementation of the core workflows of an IEEE 1609.2.1-compliant SCMS. It focuses on a "minimum complete" set of building blocks that enables real-world use while keeping the architecture clear and extensible.

OpenSCMS follows a microservices approach, where each SCMS component is isolated and can scale independently. It uses a message-driven architecture (e.g., task queues such as MQTT) to parallelize certificate requests and cryptographic operations. The backend is primarily written in Rust for safety, concurrency, and performance.

At the core is oscms-codecs-bridge, a low-level C library implementing the critical cryptographic and protocol logic, including IEEE 1609.2.1 SPDU/ASN.1 encoding/decoding, signature verification, request decryption, response construction, and pseudonym certificate generation.

The bridge is codec-agnostic: it abstracts the ASN.1 toolchain behind a clean API, reducing dependency on proprietary tooling and improving auditability and portability. It can also be used independently of the Rust backend.

By providing an open, extensible, and production-oriented SCMS platform, OpenSCMS supports faster industrial prototyping and enables academic experimentation (e.g., post-quantum transitions, misbehavior reporting, and revocation strategies), while also serving as a practical reference backend for V2X client testing.
