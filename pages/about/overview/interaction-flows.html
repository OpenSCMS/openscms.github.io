Background

Core SCMS Interaction Flows

IEEE 1609.2.1 defines two foundational end-entity–SCMS workflows:

Enrollment Certificate Provisioning

An end entity first obtains an enrollment certificate (typically from an ECA, optionally via a DCM). This establishes trust and is used for secure communication with SCMS services.

Authorization Certificate Provisioning

After enrollment, the end entity requests authorization (pseudonym) certificates used in V2X applications. Requests go through the RA (optionally via a LOP for location privacy) and are fulfilled via coordinated interaction with components such as the ACA and Linkage Authorities (LA1/LA2), ensuring that no single party can link certificates back to a specific device.
