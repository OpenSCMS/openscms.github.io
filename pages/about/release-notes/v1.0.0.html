Release Notes

OpenSCMS v1.0.0

1. Overview

OpenSCMS v1.0.0 marks the first stable release of the OpenSCMS platform. This release delivers a fully functional, standards-aligned implementation of the core server-side components defined by IEEE 1609.2.1, with a strong focus on correctness, interoperability, and extensibility.

2. Standards Compliance

  • IEEE 1609.2.1 protocol support (server-side scope)
  • Explicit and implicit certificate handling
  • ASN.1 SPDU encoding and decoding aligned with the standard

3. Core Components Included

  • Registration Authority (RA)
  • Enrollment Certificate Authority (ECA)
  • Authorization Certificate Authority (ACA)
  • Linkage Authority (LA)
  • Certificate and trust distribution services (CTL, CCF, CRL, CMIS)

4. Cryptographic Capabilities

  • Explicit and implicit certificate issuance
  • Butterfly key expansion mechanisms:
    • Original Butterfly Keys (OBK)
    • Unified Butterfly Keys (UBK)
    • Compact Unified Butterfly Keys (CUBK)
  • Encrypted and non-encrypted butterfly flows
  • ECDSA signatures and ECIES-based encryption
  • P-256 and Brainpool P-384 curve support where applicable

5. oscms-codecs-bridge Library

  • Standalone C library providing cryptographic core and ASN.1 codecs
  • Codec-agnostic architecture with ASN.1 abstraction layer
  • Integrated with Rust services using bindgen
  • Extensive unit test coverage validating cryptographic correctness

6. API and Service Interfaces

  • REST-based APIs for RA and ECA components
  • Asynchronous request processing using message queues (RabbitMQ)
  • OpenAPI / Swagger documentation available for exposed endpoints

7. Deployment and Operations

  • Unified deployment using Docker, Kubernetes, and Skaffold
  • Minikube-based local development and testing support
  • Helm charts for orchestration of services
  • Manual deployment supported for individual components

8. Configuration and Extensibility

  • Environment-variable–based configuration model
  • Fully documented deployment parameters
  • Modular architecture enabling future extensions and integrations

9. Testing and Validation

  • 120+ unit tests for the C-based cryptographic core
  • Rust unit tests covering service logic and integration layers
  • End-to-end validation using a certified V2X client
  • Validation of registration, enrollment, authorization, and download flows

10. Next Steps

Future releases will focus on expanding protocol coverage, improving operational tooling, and incorporating feedback from early adopters. Contributions, integrations, and reviews from the community are welcome.
