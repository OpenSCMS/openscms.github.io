Registration Authority (RA)

Authorization Certificate Provisioning Flow

The Authorization Certificate provisioning flow is responsible for issuing batches of authorization (pseudonym) certificates that are used by V2X clients for authenticated application-layer communication.

This flow is intentionally split into two phases: a request phase and a download phase. The request phase is synchronous and lightweight, allowing the RA to validate and acknowledge the request quickly. The actual certificate generation is performed asynchronously to support high request volumes and batch-based issuance. The download phase allows the client to retrieve the generated certificates once processing has completed.

From an architectural perspective, the RA Frontend terminates the client connection, validates and decrypts the request, and enqueues a task for asynchronous execution. One of the RA Worker instances then processes the task by invoking the Authorization Certificate Authority (ACA), persists the generated certificates, and makes them available for download.

Authorization Certificate Request

The client submits an Authorization Certificate Request SPDU to the RA Frontend. The request contains cryptographic material and authorization parameters required to generate one or more authorization certificates.

POST /v3/authorization-certificate

Description:
  Submits a request for one or more authorization (pseudonym)
  certificates.

Input:
  - IEEE 1609.2.1 Authorization Certificate Request SPDU
  - Contains enrollment reference, butterfly key parameters,
    certificate permissions, and validity constraints

Output:
  - Acknowledgment SPDU (ACK)
  - Includes timing information indicating when the client
    should issue the download request

Errors:
  - Invalid or malformed SPDU
  - Signature verification failure
  - Decryption failure
  - Policy or parameter violations

Authorization Certificate Download

GET /v3/authorization-certificate

Description:
  Retrieves the batch of authorization certificates generated
  for a previously submitted request.

Input:
  - Download Request SPDU
  - Includes request hash returned during the request phase

Output:
  - Response SPDU containing a batch of authorization
    (pseudonym) certificates

Errors:
  - Unknown or expired request hash
  - Certificates not yet available
