Registration Authority (RA)

Successor Enrollment Certificate Flow

The Successor Enrollment Certificate flow allows an already enrolled End Entity to obtain a new enrollment certificate while preserving sensitive identity information. This flow is used to renew enrollment credentials without re-running the initial enrollment process.

As with authorization provisioning, the flow is divided into a request phase and a download phase. The request phase validates and decrypts the incoming enrollment data and schedules certificate generation asynchronously. The Enrollment Certificate Authority (ECA) performs the actual issuance of the new enrollment certificate, which is then stored by the RA and made available for download.

The RA acts as the cryptographic ingress point for this flow, decrypting the request and enforcing lifecycle and policy constraints before forwarding it to the ECA.

Successor Enrollment Certificate Request

POST /v3/successor-enrollment-certificate

Description:
  Submits a request for a successor enrollment certificate.

Input:
  - Encrypted IEEE 1609.2.1 Successor Enrollment Request SPDU
  - Contains identity-related enrollment parameters

Output:
  - Acknowledgment SPDU (ACK)
  - Indicates when the client should issue the download request

Errors:
  - Decryption failure
  - Invalid enrollment state
  - Malformed request SPDU

Successor Enrollment Certificate Download

GET /v3/successor-enrollment-certificate

Description:
  Retrieves the successor enrollment certificate generated
  for a previous request.

Input:
  - Download Request SPDU
  - Includes request hash

Output:
  - Response SPDU containing the new enrollment certificate

Errors:
  - Unknown request hash
  - Certificate not yet available
