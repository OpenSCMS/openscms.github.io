Registration Authority (RA)

The Registration Authority (RA) is the primary front-end component of OpenSCMS and the main interaction point for V2X clients. For architectural simplicity, the RA consolidates the roles of the Device Configuration (DC) and Device Configuration Manager (DCM) defined in IEEE 1609.2.1.

In practice, this makes the RA responsible for End Entity (EE) lifecycle management, distribution of trust and configuration artifacts, and coordination of certificate provisioning workflows with other SCMS components.

Internal Organization and Execution Model

The RA is internally organized into two deployment-time roles: the RA Frontend and the RA Worker. The RA Frontend exposes the external REST interface and handles all client-facing protocol logic, while RA Workers execute SCMS operations asynchronously.

The RA Frontend validates incoming SPDUs, verifies signatures, decrypts encrypted payloads when required, and constructs internal tasks that are enqueued for processing. It responds immediately to the client with an acknowledgment SPDU indicating when the client should retry the corresponding download request.

RA Workers monitor the task queue and execute tasks independently. They interact with internal SCMS services such as the ACA and ECA, perform cryptographic processing, and persist results to the database using the request hash as the primary correlation key.
