Opaque vs. Exposed Structs

  • Opaque types are forward-declared in public headers, with definitions hidden in private headers or source files. This ensures ABI stability and encapsulation.
  • Exposed types are fully defined in public headers when their layout must be known to callers (e.g., OscmsSequence).
// Opaque (declared but not defined publicly)
typedef struct OscmsIeee1609Dot2Data OscmsIeee1609Dot2Data;

// Exposed (definition visible)
typedef struct {
    size_t count;
    void **data;
} OscmsSequence;

Struct Initialization

Initialize structs using designated initializers when possible to make initialization explicit and maintainable:

OscmsPoint point = {
    .x = 0x01,
    .y = 0x02
};
