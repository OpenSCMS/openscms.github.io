Scoping

Namespaces

C does not have namespaces, so function and type names use prefixes to organize scope:

  • All public API functions and types are prefixed with Oscms or oscms_.
  • Internal symbols are either declared static or prefixed with the module name to avoid symbol collisions (e.g., codec_internal_parse_point()).

Internal Linkage

Helper functions, constants, and types used only within a single translation unit should be declared static to limit their scope:

static int parse_internal_type(const uint8_t *buffer) {
    // ...
}

Local and Static Variables

Declare variables in the smallest scope possible, ideally at first use. Static variables are used sparingly and never for mutable state unless explicitly required by an initialization context.
