Build & Structure

Repository Structure

oscms-codecs-bridge/
├── codecs/                  # Codec implementations and abstractions
│   ├── asn1c/                # asn1c-based IEEE 1609.2.1 codec
│   │   ├── src/              # Generated and handwritten codec glue
│   │   └── submodules/api/   # Abstract codec API
├── include/public/           # Public bridge headers
├── src/                      # Protocol and cryptographic bridge logic
├── scms_manager/             # Utility for certificate/key generation
├── tests/                    # Unit and integration tests
└── CMakeLists.txt

The repository is organized to clearly separate codec-specific implementations from the generic protocol and cryptographic logic. This structure supports future extensibility, allowing alternative ASN.1 transpilers or codecs to be integrated without disrupting existing functionality.

Build and Installation

The oscms-codecs-bridge library is built using CMake and follows a standard out-of-source build process.

Build Steps

cd oscms-codecs-bridge
mkdir build
cd build
cmake ..
make
make install

To install into a custom directory:

make DESTDIR=install install

Running Unit Tests

The library includes an extensive suite of unit tests covering cryptographic primitives, ASN.1 encoding/decoding, and protocol message handling.

make test
