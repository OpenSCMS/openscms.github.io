Library Utilities

Memory Management Utilities

To ensure consistent and safe memory handling across the library, the API defines shared abstractions for buffers and sequences.

Octet Buffers

The OscmsOctetBuffer type encapsulates a pointer-length pair and provides a unified interface for allocation, duplication, copying, and deallocation. This avoids error-prone handling of raw pointers and lengths.

OscmsOctetBuffer *oscms_octet_buffer_new_from_buffer(
    const uint8_t *buffer, size_t length);

void oscms_free_octet_buffer(OscmsOctetBuffer *buffer);

Sequences

The OscmsSequence type provides a dynamic array abstraction similar to ASN.1 SEQUENCE OF. It is used to represent ordered collections of protocol elements while centralizing allocation and cleanup.

typedef struct OscmsSequence {
    size_t count;
    void **data;
} OscmsSequence;

Cryptographic Primitives

All cryptographic operations are implemented within the bridge layer using OpenSSL primitives. These include:

  • ECDSA signing and verification,
  • ECIES encryption and decryption,
  • SHA-256 and SHA-384 hashing,
  • implicit certificate key reconstruction,
  • elliptic curve point compression and expansion.

For example, message signing is exposed via a simple API:

int ec_sign_message(
    const OscmsOctetBuffer *message,
    EVP_PKEY *private_key,
    OscmsHashAlgorithm hash_algorithm,
    uint8_t *signature);

These primitives are used uniformly by all SCMS components, ensuring consistent cryptographic behavior across the system.

Logging and Diagnostics

The library provides a lightweight logging abstraction that allows callers to integrate with their preferred logging system or fall back to standard output or syslog.

void oscms_log_set_logger(
    oscms_log_function_ptr func,
    int log_level);

void oscms_log(int level, const char *fmt, ...);

This allows the Rust backend to redirect all C-level logs into its own structured logging pipeline.

Integration with Rust

Integration with the Rust backend is achieved through the oscms_bridge Rust crate, which compiles the C library and generates bindings at build time using bindgen. This makes the C API available as safe Rust interfaces, while preserving the performance and security properties of the native implementation.

From the perspective of Rust service developers, all cryptographic and ASN.1 processing is accessed via normal Rust function calls, without requiring knowledge of the underlying C implementation.
