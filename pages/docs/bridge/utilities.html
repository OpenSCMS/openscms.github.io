Utilities

Cryptographic Primitives

The library implements cryptographic primitives required by IEEE 1609.2.1, including:

  • Elliptic curve operations: Support for NIST P-256 and Brainpool curves.
  • Signature generation and verification: ECDSA signing and verification using specified curves.
  • Key derivation: KDF functions for deriving symmetric keys from shared secrets.
  • Hashing: SHA-256 and SHA-384 implementations.
  • Symmetric encryption: AES-128-CCM for encrypting SPDUs.

These primitives are implemented using OpenSSL and follow best practices for secure cryptographic operations.

Memory Management Utilities

The library provides a set of memory management utilities to ensure safe and consistent allocation and deallocation across the Rust-C boundary.

Key Utilities

  • Allocation wrappers: Custom allocation functions ensure compatibility between Rust and C memory models.
  • Cleanup routines: Functions for deallocating complex ASN.1 structures and cryptographic key material.
  • Memory pools: Optional pooling mechanisms for frequently allocated types.

All exported functions that allocate memory follow a consistent pattern: ownership is clearly documented, and corresponding cleanup functions are provided.

Integration with Rust

The C library is integrated into the Rust-based SCMS services using bindgen, which automatically generates Rust FFI bindings from the C header files.

Integration Approach

  • Automatic binding generation: bindgen parses the public C headers and generates corresponding Rust types and function declarations.
  • Safe wrappers: A thin Rust wrapper layer provides safe, idiomatic Rust interfaces over the raw FFI bindings.
  • Memory safety: Ownership and lifetimes are carefully managed to ensure no memory leaks or use-after-free bugs.
  • Error handling: C error codes are converted to Rust Result types for consistent error handling.

This integration approach allows Rust developers to work with the cryptographic and protocol functionality as if it were native Rust code, while benefiting from the performance and security characteristics of the C implementation.

Logging and Diagnostics

The library includes a logging and diagnostics framework to aid in debugging and operational monitoring.

Features

  • Structured logging: Log messages include context such as function name, file, and line number.
  • Log levels: Support for debug, info, warning, and error levels.
  • Integration with Rust: Logs generated in C can be forwarded to Rust's logging infrastructure.
  • Diagnostic utilities: Functions for dumping ASN.1 structures, cryptographic keys, and protocol messages in human-readable formats.

Logging can be configured at runtime to control verbosity and output destinations.
