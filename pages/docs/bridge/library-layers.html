Library Layer Design

Codec Abstraction Layer

The codec abstraction layer defines a stable API that isolates the rest of the library from codec-specific data structures generated by ASN.1 transpilers. This API is located under:

codecs/submodules/api/

Concrete codec implementations (currently based on asn1c) implement this API and handle conversion between internal ASN.1 representations and abstract data structures exposed to the rest of the library.

For example, the ASN.1 representation of an elliptic curve point is defined by the transpiler as a deeply nested union-based structure. The codec API exposes an abstract and codec-independent representation:

ASN.1 Internal Representation (simplified)

typedef struct EccP256CurvePoint {
    ...
    union {
        OCTET_STRING_t x_only;
        struct {
            OCTET_STRING_t x;
            OCTET_STRING_t y;
        } uncompressedP256;
    } choice;
} EccP256CurvePoint_t;

Abstract API Representation

typedef struct OscmsEccCurvePoint {
    OscmsEccPointCurveType curve_type;
    OscmsEccPointType point_type;
    OscmsOctetBuffer x;
    OscmsOctetBuffer y;
} OscmsEccCurvePoint;

Conversion functions, such as oscms_ecc_curve_point_from_internal, translate between codec-specific and abstract representations. This pattern is consistently applied across all ASN.1-derived types.

Protocol Bridge Layer

The bridge layer implements the semantics of IEEE 1609.2.1 protocol messages and server-side flows. It provides functionality for:

  • constructing and parsing SPDUs,
  • signing and verifying messages,
  • encrypting and decrypting payloads,
  • validating certificate chains and trust anchors.

This layer is completely codec-agnostic and operates exclusively on abstract types provided by the API layer.
