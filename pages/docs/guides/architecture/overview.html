Architecture

Overview

OpenSCMS follows a layered and service-oriented architecture aligned with the logical decomposition proposed by IEEE 1609.2.1. Each SCMS component is implemented as an independent Rust crate following microservices design, allowing clear separation of responsibilities, independent scaling, and reduced trust concentration. Based on that, V2X end entities (primarily OBUs and RSUs) can interact with the SCMS via IEEE 1609.2.1-defined REST interfaces.

High-Level Layers

The system can be viewed as five primary layers:

    • REST API Layer (Rust): Rust-based microservices implementing SCMS components (RA, ECA, ACA, LA) and exposing the external REST API.
    • Rust–C Bridge Layer (Rust): Interoperability layer that connects Rust code to the C-based oscms-codecs-bridge library, handling safe data marshalling between the two runtimes.
    • Protocol and Cryptography Bridge Layer (C): The oscms-codecs-bridge C library, responsible for multiple critical functions: handling ASN.1-based SPDUs (Secure Protocol Data Units), performing encryption/decryption, and orchestrating protocol-level cryptographic operations.
    • Codec Abstraction Layer (C): A well-defined C API that abstracts the underlying codecs, providing a uniform interface for encoding/decoding and decoupling protocol logic from codec implementations.
    • Codec Implementation Layer (C): The concrete low-level implementations of the ASN.1 codecs used by the system, focused on efficient parsing, serialization, and performance-critical operations.
