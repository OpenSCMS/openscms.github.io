Internal Structure of a Service Crate

Although each service implements a different SCMS role, their internal structure follows a common pattern, which simplifies navigation and onboarding:

  • main.rs: Service entry point, responsible for startup, configuration loading, and HTTP server initialization.
  • endpoint.rs: Definition of REST or binary endpoints exposed by the service.
  • core_logic/: Implementation of the service's domain-specific logic, including certificate workflows and protocol handling.
  • entities/: Data models representing persisted or derived SCMS entities.
  • persistence/: Database access layer and query abstractions.
  • migrator/: Schema migrations required by the service.

This separation enforces a clear distinction between transport and API concerns, protocol and cryptographic logic, and persistence and state management.
