The oscms_bridge Rust Crate

The oscms_bridge crate is a Rust library crate that provides a safe and idiomatic Rust interface to the C-based oscms-codecs-bridge. It serves as the explicit language and responsibility boundary between Rust services and the low-level cryptographic and protocol engine.

This crate contains a build.rs script responsible for:

  • Compiling the C sources of the oscms-codecs-bridge
  • Linking the resulting static or shared library into the Rust build
  • Automatically generating Rust bindings for the C API using the bindgen tool

By generating bindings at build time, oscms_bridge ensures that the Rust view of the C API remains consistent with the underlying implementation, while preserving Rust's safety guarantees through carefully designed wrapper types and ownership boundaries.

The oscms_bridge crate also contains the oscms-codecs-bridge subrepository, which is itself a fully independent C project. This C component is structurally and conceptually isolated from the Rust workspace and implements the core cryptographic and IEEE 1609.2.1 protocol logic of OpenSCMS.

The oscms-codecs-bridge is built using CMake and does not depend on Rust code or Rust-specific tooling. It exposes a well-defined C API that is consumed by multiple SCMS services via the oscms_bridge Rust lib crate, but remains entirely agnostic to service-level concerns and deployment details.

This strict separation allows the C bridge to be developed, audited, and reused independently of the Rust backend. The design, build system, and coding guidelines of the oscms-codecs-bridge are described in detail in a dedicated section later in this document.
