Setup & Deployment

Certificates and Keys

All SCMS components require an IEEE 1609.2 explicit certificate and corresponding key pairs to participate in the system. These certificates and keys are used during the communication proccess between v2x client and SCMS signing operations, and enrollment processes. Each service must have access to its certificate and private key before it can be started.

OpenSCMS provides a certificate generation script for development and testing purposes. This script generates all cryptographic payloads required by openSCMS services, including component certificates, public and private keys, and protocol-specific artifacts. It also produces external trust material used by V2X clients, such as elector certificates and Root CA keys.

Generating Certificates for Development

From the repository root, run the certificate generation script:

./scripts/run_test_scms_manager.sh

This script generates a full set of test certificates and stores them in the certs/ directory within the repository. Each service will look for its certificate and key in this directory by default.

External Service Files

In addition to internal SCMS component certificates, the scms_manager generates external trust artifacts that are not consumed directly by openSCMS services but are essential for V2X clients.

These files include elector certificates and Root CA material, which together form part of the V2X client root of trust:

external_service_files/
                        ├── electors_files
                        │   ├── 1_electorCertificate.coer
                        │   ├── 1_electorPrivateKey.dat
                        │   ├── 1_electorPublicKey.dat
                        │   ├── 2_electorCertificate.coer
                        │   ├── 2_electorPrivateKey.dat
                        │   ├── 2_electorPublicKey.dat
                        │   ├── 3_electorCertificate.coer
                        │   ├── 3_electorPrivateKey.dat
                        │   └── 3_electorPublicKey.dat
                        ├── ica_files
                        │   ├── icaCertificate.coer
                        │   ├── icaPrivateKey.dat
                        │   └── icaPublicKey.dat
                        └── root_ca_files
                            ├── rootcaCertificate.coer
                            ├── rootcaPrivateKey.dat
                            └── rootcaPublicKey.dat

Important: The certificates generated by this script are intended for development and testing only. They should never be used in production environments. Production deployments must use properly issued certificates from trusted Certificate Authorities.

Custom Certificate Configuration

For production deployments, custom certificates must be generated and configured. The expected certificate paths can be overridden using environment variables specific to each service. Refer to each service's README.md for details on configuring custom certificate paths.
