Setup & Deployment

Manual Deployment

Manual deployment allows individual SCMS services to be built and run directly on a host system without containerization. This approach is particularly useful during development and debugging, as it provides direct access to logs, processes, and configuration files. However, it requires more careful configuration than unified Kubernetes-based deployments.

This section describes the process of manually deploying a single service, using the Registration Authority (RA) as an example. The same general workflow applies to all other services (LA, ECA, ACA).

Prerequisites

Before deploying any service, ensure that:

  • All system dependencies have been installed (see the "Build Dependencies" section).
  • A MySQL server is running and accessible.
  • A RabbitMQ broker is running and accessible.
  • Service certificates and keys have been generated and are present in the certs/ directory.

Example: Deploying the Registration Authority (RA)

Setting Up Environment Variables

Each service requires environment variables to configure database connections, message broker URLs, certificate paths, and API ports. For the RA service, the following environment variables must be defined:

MYSQL_USERNAME=username
MYSQL_PASSWORD=password
MYSQL_HOST=mysql
MYSQL_PORT=3306
MYSQL_CONNECTION_STRING=mysql://username:password@mysql:3306

RABBITMQ_CONNECTION_STRING=amqp://rmqadmin:password@scmsrabbitmq:5672

Building and Running the RA Service

Once the environment is correctly configured, the RA service can be built and started.

From the repository root, build the complete Rust workspace:

cargo build

This step compiles all Rust service crates as well as the C-based oscms-codecs-bridge, and generates the required Rust bindings via bindgen. Build failures at this stage typically indicate missing system dependencies or misconfigured toolchains.

To start the Registration Authority service, change into the RA crate directory and run:

cd openscms/ra
cargo run

Upon startup, the RA service will load its certificates and keys, establish connections to the configured MySQL database and RabbitMQ broker, and expose its REST endpoints on the configured port.

Startup failures usually result from missing certificates, incorrect environment variables, or unavailable external services.
