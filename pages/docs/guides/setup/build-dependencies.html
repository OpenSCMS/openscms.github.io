Setup & Deployment

Build Dependencies

Before building and deploying OpenSCMS, it is important to ensure that all necessary dependencies are installed on the system. OpenSCMS has a layered architecture that combines multiple technologies, and each layer requires specific tooling to be present.

Core Dependencies for Manual Deployment

To compile and run OpenSCMS services manually (without Docker), the following system libraries and tools are required:

  • Rust Toolchain: OpenSCMS services are written in Rust and require a working Rust installation. Install Rust via rustup:
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
rustup update
  • C Compiler and Build Tools: The oscms-codecs-bridge integrates C-based cryptographic libraries. On macOS, install Xcode command-line tools:
xcode-select --install

On Linux systems, ensure gcc, make, and other standard build tools are available.

  • CMake: Required for building the oscms-codecs-bridge C extension. Install via Homebrew on macOS:
brew install cmake

On Linux, use the system package manager (e.g., apt install cmake on Debian-based distributions).

  • libclang: The Rust bindgen tool requires libclang to generate Rust bindings for C headers. On macOS:
brew install llvm

On Linux:

apt install libclang-dev
  • MySQL Client Libraries: Services that interact with MySQL (such as the Registration Authority) require MySQL development headers. On macOS:
brew install mysql-client

On Linux:

apt install libmysqlclient-dev
  • RabbitMQ: For local development, a running RabbitMQ instance is required. Install via:
brew install rabbitmq

Or via Docker:

docker run -d --name rabbitmq -p 5672:5672 -p 15672:15672 rabbitmq:management

Unified Deployment Dependencies

For containerized deployments using Docker and Kubernetes, the following tools are required:

  • Docker: Required to build and run container images. Install Docker Desktop on macOS or Docker Engine on Linux.
  • Minikube: Provides a local Kubernetes cluster for development and testing.
  • Skaffold: Orchestrates the build and deployment pipeline.
  • kubectl: The Kubernetes command-line interface.
  • Helm: Used to deploy certain infrastructure components such as RabbitMQ operators.

OpenSCMS recommends managing all Kubernetes tooling through binenv, a language-agnostic tool version manager that simplifies installation and ensures consistent versions across different machines.

Installing binenv

To install binenv on macOS or Linux:

curl -sL https://github.com/devops-works/binenv/releases/latest/download/binenv-$(uname -s)-$(uname -m) \
  -o /usr/local/bin/binenv && chmod +x /usr/local/bin/binenv

Initialize binenv:

binenv update

Add binenv to your shell's PATH. For Zsh, add the following to ~/.zshrc: 

export PATH="$HOME/.binenv:$PATH"

For Bash, add the equivalent line to ~/.bashrc.

Installing Required Tools via binenv

With binenv installed, install the required Kubernetes tooling:

binenv install minikube
binenv install skaffold
binenv install kubectl
binenv install helm

Verify that each tool is correctly installed:

minikube version
skaffold version
kubectl version --client
helm version

These tools form the complete foundation required to deploy OpenSCMS in a containerized environment.
