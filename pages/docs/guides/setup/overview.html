Setup & Deployment

Overview

OpenSCMS supports multiple deployment options to accommodate different operational requirements. The system has been designed with flexibility in mind, and components can run either directly on host machines or in containerized environments such as Docker and Kubernetes. This section provides guidance on setting up dependencies, generating cryptographic certificates, and deploying SCMS services both manually and using container orchestration.

For most operational use cases, a unified Kubernetes-based deployment is recommended, as it provides the most streamlined setup experience and ensures consistency across development, testing, and production environments. However, for debugging, development, and deeper architectural analysis, manual deployment of individual components remains fully supported.
