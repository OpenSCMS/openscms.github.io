The OpenSCMS

The OpenSCMS project is an open-source implementation of a Security Credential Management System (SCMS) compliant with the IEEE 1609.2 and IEEE 1609.2.1 standards for V2X communication.

Unlike traditional PKI systems, an SCMS must simultaneously satisfy strict cryptographic correctness, large-scale performance, and strong privacy requirements. In particular, the SCMS must enable authenticated and authorized communication between V2X end entities while preventing long-term tracking of vehicles and infrastructure nodes.

The OpenSCMS addresses this challenge by providing a "minimum complete" implementation of the core SCMS workflows defined by IEEE 1609.2.1. The system supports enrollment certificate provisioning and authorization certificate provisioning, including butterfly key expansion to compose pseudonym certificate generation.

Engineering Perspective

From an engineering perspective, OpenSCMS is designed as a distributed, microservice-based system implemented primarily in Rust, with a dedicated cryptographic and protocol engine implemented in C. This separation allows the system to combine strong safety and concurrency guarantees at the service level with precise control over cryptographic operations and ASN.1 codecs operation at the protocol level.

Documentation Purpose

This technical documentation complements the OpenSCMS White Paper by focusing on implementation details, repository structure, deployment procedures, APIs, and contribution guidelines, with the goal of enabling engineers to understand, execute, extend, and contribute to the OpenSCMS codebase.
