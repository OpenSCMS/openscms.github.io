SCMS Components

Overview of the main components that make up the Security Credential Management System.

Registration Authority (RA)

The Registration Authority is responsible for handling enrollment and registration of devices in the SCMS ecosystem.

Key Functions:

  • Device enrollment processing
  • Identity verification
  • Certificate request handling
  • Enrollment certificate issuance coordination

Linkage Authority (LA)

The Linkage Authority manages linkage values essential for privacy preservation while maintaining the ability to revoke misbehaving devices.

Key Functions:

  • Linkage value generation and distribution
  • Privacy-preserving certificate linking
  • Revocation support

Authorization Certificate Authority (ACA)

The ACA issues short-lived authorization certificates used for V2X communications.

Key Functions:

  • Authorization certificate generation
  • Certificate distribution to enrolled devices
  • Certificate lifecycle management

Enrollment Certificate Authority (ECA)

The ECA issues enrollment certificates that devices use to request authorization certificates.

Key Functions:

  • Enrollment certificate issuance
  • Device identity binding
  • Long-term credential management

Misbehavior Authority (MA)

The MA handles detection, reporting, and response to misbehaving devices in the system.

Key Functions:

  • Misbehavior report processing
  • Device revocation
  • Security incident response

System Overview

OpenSCMS implements a distributed architecture where components communicate via REST APIs, enabling scalable and flexible deployment.

Component Interaction

Components interact through well-defined APIs following the IEEE 1609.2.1 standard specifications.
