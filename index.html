OpenSCMS

An open-source, deployable Security Credential Management System (SCMS) aligned with IEEE 1609.2.1 (2022)—built for modularity, observability, and real-world scale.

RA / ECA / ACA services Docker + Kubernetes OpenAPI specs 1609.2-style (non-X.509) certs

Key Capabilities

OpenSCMS implements core SCMS workflows with a focus on efficiency, scalability, and deployability. It supports provisioning certificates for both OBU and RSU clients, including pseudonym certificate issuance.

◆ Standards-Aligned APIs

Implements IEEE 1609.2.1 endpoint semantics per component, published as OpenAPI v3 specs.

◆ High-Throughput Workflows

Uses task parallelism and asynchronous processing where required (e.g., authorization certificate generation).

◆ OBU / RSU Provisioning

OBUs can request pseudonym or identification/application certificates. RSUs can request identification certificates.

◆ Implicit & Explicit Pseudonyms

Supports implicit (EC25V, smaller payloads) and explicit (ECDSA P-256) pseudonym certificates.

Deployment & Ops

  • Docker + Kubernetes deployment (local Minikube or production clusters)
  • Operational features: logging, parameter customization, and fault recovery mechanisms
  • Auxiliary scripts for bootstrapping artifacts: CTL, CCF payloads, elector/root CA keys

Designed to be deployable, observable, and scalable across different deployment sizes.

API Highlights

  • RA: registration, authorization cert request/download, CTL/CCF, CRLs, CA cert download
  • ECA: enrollment cert issuance, successor enrollment responses
  • ACA: authorization support operations

Many payloads are exchanged as binary encodings aligned with IEEE 1609.2.1 conventions.

Architecture at a Glance

Each component maintains its own state (local database). Public APIs are exposed per role and described via OpenAPI.

Additional Highlights

Certificates

  • OBU: pseudonym + identification/app
  • RSU: identification
  • Implicit (EC25V) & explicit (ECDSA P-256)
  • 1609.2 format (non-X.509)

Designed for modularity: the ASN.1 codec layer is independent and can be swapped for alternative implementations.

Independent ASN.1 Codec

OpenSCMS uses an independent ASN.1 codec library for IEEE 1609.2 encoding/decoding that will be released as open source.
